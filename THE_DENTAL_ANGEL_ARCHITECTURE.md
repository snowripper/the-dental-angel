## 2.1 Data Flows

### Chat Flow
```
User types question
       │
       ▼
App sends to Edge Function (/api/chat)
       │
       ▼
Edge Function:
  1. Check rate limit (free tier: 3/month)
  2. Retrieve context from pgvector (RAG)
  3. Build prompt with Dr. Angel personality
  4. Calculate Second Opinion Score
  5. Call OpenAI GPT-4o
  6. Stream response back
       │
       ▼
App displays streaming response
       │
       ▼
Save conversation to database
```

### Photo Upload Flow (Treatment Plan OCR)
```
User takes photo of treatment plan
       │
       ▼
Upload to Supabase Storage
       │
       ▼
Edge Function:
  1. Retrieve image
  2. Send to OpenAI Vision API
  3. Extract: procedures, codes, costs
  4. Run through Dental Dialect Translator
  5. Generate explanations for each item
  6. Calculate Second Opinion Score
  7. Store parsed treatment plan
       │
       ▼
Display breakdown to user
```

### Family Share Flow (Viral Loop)
```
User taps "Share"
       │
       ▼
Generate unique share token
       │
       ▼
Create shareable URL: dental-angel.com/share/{token}
       │
       ▼
Recipient opens link (no app needed)
       │
       ▼
Server renders formatted explanation
       │
       ▼
Track view, prompt app download
```

-- Function to match dental knowledge using vector similarity
CREATE OR REPLACE FUNCTION public.match_dental_knowledge(
  query_embedding VECTOR(1536),
  match_threshold FLOAT DEFAULT 0.7,
  match_count INT DEFAULT 5
)
RETURNS TABLE (
  id UUID,
  procedure_code TEXT,
  title TEXT,
  content TEXT,
  plain_explanation TEXT,
  questions_to_ask JSONB,
  similarity FLOAT
) AS $$
BEGIN
  RETURN QUERY
  SELECT
    dk.id,
    dk.procedure_codes[1] AS procedure_code,
    dk.title,
    dk.content,
    dk.plain_explanation,
    dk.questions_to_ask,
    1 - (dk.embedding <=> query_embedding) AS similarity
  FROM public.dental_knowledge dk
  WHERE dk.embedding IS NOT NULL
    AND 1 - (dk.embedding <=> query_embedding) > match_threshold
  ORDER BY dk.embedding <=> query_embedding
  LIMIT match_count;
END;
$$ LANGUAGE plpgsql;

-- Function to reset monthly question count
CREATE OR REPLACE FUNCTION public.reset_monthly_questions()
RETURNS void AS $$
BEGIN
  UPDATE public.profiles
  SET questions_this_month = 0,
      month_reset_at = DATE_TRUNC('month', NOW()) + INTERVAL '1 month'
  WHERE month_reset_at <= NOW();
END;
$$ LANGUAGE plpgsql;

- Always acknowledge the patient's emotions and concerns

- Admits uncertainty when appropriate

- Use phrases like "typically", "commonly", "often" - never absolutes
- Frame responses as "here's what to understand" and "here's what to ask"
- End responses with "Questions to ask your dentist" when appropriate

Your features:
- Second Opinion Score: Rate confidence (0-100) that a treatment is typically necessary for described situations
- Dental Dialect Translator: Convert complex jargon to plain English
- Questions Generator: Create specific questions for their dentist

Knowledge includes:

- Understanding of treatment plan formats and ADA codes
- Cost ranges and factors that affect pricing
- What to expect before, during, and after procedures
- Common patient concerns and how to address them

Example tone:
"Ah, a crown! Let me explain what's going on there. Think of it like a protective helmet for your tooth - it covers and protects a tooth that's been damaged or weakened. Now, there are several reasons your dentist might recommend one..."

    if (profile?.subscription_tier === 'free') {
      await supabase
        .from('profiles')
        .update({ questions_this_month: profile.questions_this_month + 1 })
        .eq('id', userId)
    }

    return new Response(JSON.stringify({
      response,
      secondOpinionScore
    }), {
      headers: { 'Content-Type': 'application/json' }
    })

  } catch (error) {
    return new Response(JSON.stringify({ error: error.message }), { status: 500 })
  }
})

async function getEmbedding(text: string): Promise<number[]> {
  const response = await openai.embeddings.create({
    model: 'text-embedding-ada-002',
    input: text
  })
  return response.data[0].embedding
}

function calculateSecondOpinionScore(
  question: string,
  response: string,
  knowledge: any[]
): number {
  // Simplified scoring logic
  // In production, this would be more sophisticated
  let score = 75 // Base score

  // Adjust based on knowledge match quality
  if (knowledge?.length > 0) {
    const avgSimilarity = knowledge.reduce((sum, k) => sum + k.similarity, 0) / knowledge.length
    score += (avgSimilarity - 0.7) * 50 // Boost for high-quality matches
  }

  // Clamp between 0-100
  return Math.max(0, Math.min(100, Math.round(score)))
}

# 7. KEY TECHNICAL DECISIONS

| Decision | Choice | Rationale |
|----------|--------|-----------|
| **Anonymous-First Auth** | Supabase anonymous | Zero friction to first value, easy upgrade path |
| **OpenAI GPT-4o** | Over Claude | Vision API for OCR, faster streaming |
| **RevenueCat** | Over direct Stripe | Handles iOS/Android complexity |
| **Pre-Generated Videos** | Over real-time | Consistent quality, no latency, cost predictable |
| **Supabase pgvector** | Over Pinecone | Keeps stack simple, good enough performance |
| **React Native + Expo** | Over Flutter | Better ecosystem, OTA updates, founder preference |
| **Second Opinion Score** | 0-100 scale | Clear, actionable confidence indicator |

npx tailwindcss init

cat > .env << EOF
EXPO_PUBLIC_SUPABASE_URL=your-url
